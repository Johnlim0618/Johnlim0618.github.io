Atomic Red Team Validation Workflow

This page shows a lightweight validation checklist you can use after running an Atomic test. It helps confirm whether the expected telemetry and detection behavior actually appeared.

What this project shows

  • Threat simulation and validation discipline
  • Telemetry verification mindset
  • Lab evidence collection

Example artifact

Validation Checklist
1. Run the Atomic Red Team test safely in the lab
2. Confirm expected Windows event logs were generated
3. Check if Sysmon / WEF forwarded the activity
4. Search Splunk / Wazuh for the related telemetry
5. Confirm whether an alert triggered
6. Document gaps, false negatives, and tuning ideas
