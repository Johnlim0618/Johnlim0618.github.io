Splunk + Wazuh Detection Lab

This project page shows a sample Splunk search that can be used to identify repeated failed logons followed by a successful authentication from the same user.

What this project shows

  • Windows log review and basic correlation logic
  • Authentication triage workflow
  • SIEM-oriented thinking around suspicious login patterns

Example artifact

index=wineventlog (EventCode=4625 OR EventCode=4624)
| eval user=coalesce(TargetUserName, Account_Name)
| stats count(eval(EventCode=4625)) as failed_count
        count(eval(EventCode=4624)) as success_count
        min(_time) as first_seen
        max(_time) as last_seen
        by user host
| where failed_count >= 5 AND success_count >= 1
| convert ctime(first_seen) ctime(last_seen)
